# 💐 独立高定花艺 Studio 全栈系统 (Monorepo)

本项目是一款专门为独立高端定制花艺工作室（Studio）量身打造的**跨端一体化全栈电商与 WMS 仓储管理系统**。项目采用现代化的 Monorepo（大仓）架构，完美闭环了从移动端顾客“沉浸式选花、下单”到 PC 运营端花艺师“流水线看板作业、逆向退款损耗控制”的完整商业生命周期。

---

## 🌟 核心业务与技术亮点

### 1. 📱 小程序 C 端：全沉浸式高定零售体验
- **一维单款式（SPU ➔ SKU）架构：** 告别传统大厂电商冗余的复杂多维矩阵，聚焦高定花艺“所见即所得”的单物理变体列表，店长录入更轻量，顾客选择更直观。
- **高保真规格弹窗（带信息联动大图预览）：** 弹窗内每个款式均挂载绝对定位的放大镜图标。点击触发全屏沉浸式遮罩层，**左上角提供显式关闭按钮，底部毛玻璃卡片灵魂联动品名与专属价格**。
- **免认证合规静默登录：** 针对个人主体（未认证）实施“数据平权”架构，利用 `wx.login` 静默置换 `OpenID` 作为唯一主键，用户全程无感加购；多页面购物车红点（Badge）在 `onShow` 中动态全量同步。
- **一键微信地址解构：** 结算页调用 `wx.chooseAddress` 一键拉起微信官方地址，自动解构收货人、电话并动态回填，完美绕过个人主体无法获取手机号的政策限制。
- **鲜花专属业务存证：** 结算页满血集成【预约配送时间选择器】与【贺卡寄语输入框】，为浪漫经济提供核心履约支撑。
- **购物车精准清洗算法：** 订单提交成功后，利用 `filter` 精准剔除本次购买的 SKU 条目，**严禁全盘擦除缓存**，完美保留用户未勾选的其他心仪商品。

### 2. 👨‍💼 CMS 管理端：Trello 风格 5 列流式拖拽订单看板
- **全局视图流式阵列：** 彻底抛弃传统扁平表格，采用大厂标准的 Kanban 看板。订单根据生命周期自动归类进入 5 个垂直池：
  `1. 待付款` ➜ `2. 待制作` ➜ `3. 花艺制作中` ➜ `4. 鲜花配送中` ➜ `5. 已完成/历史归档`。
- **单向物理拖拽流转（HTML5 Drag & Drop）：** 鼠标抓取卡片向右拖拽即可丝滑流转状态。流转至【配送中】时自动触发 Popover 强控录入骑手电话或闪送单号。**内置逆向拖拽拦截器**，严禁倒退状态。
- **状态色彩心理学归档（Color Coding）：** 第五列【已完成/历史归档】大收拢池对终点订单实施视觉硬隔离：
  - 🟢 **真正完成：** 浅绿背景 + 绿边框，代表顾客已确认收货。
  - ⚪ **顾客取消：** 浅灰背景 + 灰边框，代表待支付阶段顾客主动取消。
  - 🟤 **店长关闭：** 暗灰边框 + 强控标签，代表恶意订单被后台掐断。
  - 🔴 **退款取消：** 柔和浅红背景 + 强有力红边框，醒目显示 `(已退¥XXX)`，财务亏损精确存证。

### 3. 🔐 后端与 WMS 仓储：高并发防御与逆向损耗控制
- **原子级锁库存（并发超卖防御）：** 创建订单时利用 Prisma 事务对 `ProductSku` 执行原子减量，强加 `stock >= quantity` 乐观锁限制条件，彻底击穿高并发时间差，确保节假日爆单不超卖。
- **状态机并发冲突防御：** 所有状态修改接口（包含用户端收货、管理端流转）在 `where` 条件中强制死锁前置状态，杜绝双端并行操作引发的脏写风险。
- **逆向库存选择性回滚：** 针对鲜花高损耗属性，店长点击 `[退款并取消]` 时可动态传参 `rollbackStock`。待制作阶段一键原路退回 WMS 库存；已制作/已剪根阶段不退库存只走账目流失，精准控损。
- **动态时间防御校验：** 用户在“待支付”状态跨天唤起支付时，后端动态校验当前时间是否已超过订单指定的配送时间，超时自动拦截，防范幽灵订单死锁库存。

## 📐 数据库核心模型快照 (`prisma.schema`)

系统采用严密的【表名蛇形、字段驼峰】命名规范。核心实体包含：
- `ProductSpu (@@map("product_spus"))`: 存储公用属性及养护指南（`maintenanceGuide`）。
- `ProductSku (@@map("product_skus"))`: 存储款式、价格、物理库存及款式主图标识（`isMainImage`）。
- `User (@@map("users"))`: 存储 OpenID 及默认收货冗余快照。
- `Order (@@map("orders"))`: 驱动 6 大状态机，硬编码冗余配送时间、贺卡寄语及退款账目。
- `OrderItem (@@map("order_items"))`: 历史存证快照表，下单瞬间将商品名、款式名、价格硬编码写入，防止后续店长改价引发历史对账灾难。
